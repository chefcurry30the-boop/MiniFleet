# Default MiniFleet Playbook

## Read → Work → Verify → Write

1. **Read state** — Open `state.md` and `checklist.md` in the job workspace
2. **Do work** — Complete exactly one focused unit of work
3. **Verify** — Run tests/lint if applicable
4. **Write state** — Update `state.md` with progress and next steps

## Rules
- One iteration = one unit of work
- Document blockers in state.md
- Output `MINIFLEET_TASK_COMPLETE` only when genuinely done
